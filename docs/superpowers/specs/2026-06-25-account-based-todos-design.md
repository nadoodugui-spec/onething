# 원씽 "계정별 할 일"(per-account notebook) 설계

작성일: 2026-06-25
대상 파일: `C:\AI\OneThing\index.html`
선행: [요청 주고받기 메신저](2026-06-25-onething-requests-design.md) 구현 이후의 수정.

---

## 1. 배경 / 문제

요청 메신저를 붙이면서 로그인(계정)을 추가했지만, **할 일 데이터는 여전히 "동기화 코드" 기준**이라
같은 기기에서 계정을 바꿔도 할 일이 똑같이 보인다(나두혁·양수민이 같은 할 일을 공유). 또 로그인 화면이
반투명이라 **뒤의 할 일이 비쳐 보인다.**

사용자 요구: **할 일을 계정별로 완전히 분리**하고, 로그인 화면 뒤로 할 일이 보이지 않게 한다.

## 2. 확정된 결정 (사용자 합의)

1. **계정이 기기를 따라옴** — 어느 기기서든 같은 계정으로 로그인하면 그 계정의 할 일이 나타난다.
   클라우드 저장을 **계정 기준**(`notes/user_<userId>`)으로 바꾼다. (기존 "동기화 코드" 방식은 더 이상 동기화에 쓰지 않는다.)
2. **동기화 칩은 계속 표시** — 단, 역할이 바뀐다. 코드 입력 없이 **'계정 기준 자동 동기화' 상태만** 보여준다(녹색 "동기화 켜짐").
3. **기존 할 일 → '나두혁' 계정으로 1회 이전** — 나두혁 계정이 비어있을 때만. 양수민 등 새 계정은 빈 상태.
4. **로그인 화면 불투명** + 로그인 전 할 일 비표시.

## 3. 데이터 모델 변경

### 3.1 로컬 저장 (localStorage)
- 기존: 단일 키 `onething-notebook-v1` 하나에 `state` 저장.
- 변경: **계정별 키** `onething-notebook::<userId>`. 로그인한 계정의 키에서 읽고 쓴다.
- 헬퍼: `notebookKey()` → 로그인 상태면 `"onething-notebook::" + currentUser.id`, 아니면(미로그인·메시징 비활성 레거시) 기존 `STORAGE_KEY`.
- 마이그레이션 플래그: `onething-acct-migrated`("1"이면 이전 완료).

### 3.2 클라우드 저장 (Firebase)
- 기존: `notes/<동기화코드>` = `{data, ts, origin}` — **동기화에 사용 중단**(노드는 남겨두되 읽지 않음).
- 변경: `notes/user_<userId>` = `{data, ts, origin}`. 로그인한 계정 기준으로 구독·푸시.
- 메시징(`users/`, `requests/`)은 **그대로**(영향 없음).

## 4. 코드 흐름 변경

### 4.1 저장/로드를 계정 기준으로
- `loadState()` → `notebookKey()`에서 읽는다.
- `save()` → `notebookKey()`에 쓰고 `pushCloud()` 호출(계정 ref로).
- 타이머의 직접 `localStorage.setItem(STORAGE_KEY, ...)` (focusSec 15초 저장)와
  클라우드 pull 시 로컬 기록 부분도 `notebookKey()`를 쓰도록 교체.

### 4.2 클라우드 동기화를 계정 ref로
- 기존 `initSync()`는 `notes/<코드>`를 구독했다. 이를 **계정 ref**(`notes/user_<currentUser.id>`) 구독으로 바꾼다(`initAccountSync()`).
  - 로그인 사용자가 없으면 구독하지 않는다.
  - 로그인/계정 전환 시 기존 구독을 끊고(`syncRef.off()`) 새 계정 ref로 다시 구독한다.
  - `pushCloud()`/`syncRef`/`cloudReady`/`lastTs`/`applyingRemote`/`clientId` 기존 메커니즘 재사용(구독 대상만 변경).
- 동기화 코드 함수(`getSyncCode`/`saveSyncCode`/`genCode`)는 동기화 구동에서 분리(미사용). 삭제하지 않고 남겨두되 호출하지 않는다.

### 4.3 계정 전환 시 노트북 교체
- `switchNotebook()`:
  1. `state = loadState()` (현재 계정 키에서)
  2. `applyLang(); render();`
  3. `initAccountSync()` (계정 ref 재구독)
  4. (조건 충족 시) 마이그레이션 실행
- 호출 시점: 로그인 성공 후, 초기화에서 로그인 복원 후(`resolveCurrentUser`가 사용자 확정한 뒤), 로그아웃 시(빈 상태로).
- 로그아웃 시: `state = migrate(null)`(빈)으로 두고 render(불투명 게이트가 가림). 이전 계정 잔상 방지.

### 4.4 동기화 칩(상태 표시) 변경
- 칩은 계속 표시. `setSyncStatus`를 계정 동기화 상태에 맞춰 사용:
  - 로그인 + 클라우드 연결됨 → 녹색 "동기화 켜짐"(계정 기준).
  - 미로그인/오프라인 → 회색/경고.
- 칩 클릭 시 모달: **코드 입력·생성·복사 행을 숨기고**, "로그인한 계정 기준으로 자동 동기화됩니다" 안내만 표시.

### 4.5 로그인 게이트
- `.login-gate` 배경을 **불투명**(예: 솔리드 데스크 색)으로 변경 → 뒤 할 일 비치지 않음.
- 로그인 필수화: '그냥 둘러보기' 스킵 버튼 제거(할 일이 계정 종속이라 로그인 전엔 보여줄 것이 없음).
  - 단, 메시징 비활성(`cloudConfigured()==false`) 환경에서는 기존처럼 게이트 없이 단일 로컬 노트로 동작(레거시 호환).

## 5. 기존 할 일 마이그레이션 (1회)

규칙(데이터 안전 우선):
- 조건: `currentUser.name === "나두혁"` 그리고 `onething-acct-migrated` 미설정 그리고 **현재 계정 노트북이 비어있음**(todos·history 0) 그리고 레거시(`onething-notebook-v1`)에 내용 존재.
- 동작: 레거시 `state`를 현재 계정 노트북으로 `migrate()`해서 채우고 `save()`(로컬 키 + 클라우드 push). 플래그 설정.
- 타이밍: **계정 클라우드 구독의 첫 스냅샷이 비어있음(null)을 확인한 뒤** 실행 → 이미 클라우드에 데이터가 있는 계정을 덮어쓰지 않음.
- 안전장치: 계정에 이미 할 일이 있으면(또는 클라우드에 데이터가 있으면) 마이그레이션하지 않는다.

## 6. 예외 / 엣지

- **오프라인 + 이미 로그인된 계정**: 계정별 로컬 캐시로 계속 사용. 연결되면 push로 반영.
- **오프라인 + 미로그인 + 미기억**: 등록/로그인 불가(클라우드 필요) → 사용 보류. (인터넷 1회 필요 — 수용된 한계.)
- **계정 클라우드 충돌**: 동일 계정을 두 기기서 동시 수정 시 기존처럼 타임스탬프 last-write-wins.
- **레거시 동기화 코드 데이터**: `notes/<코드>`는 그대로 남지만 읽지 않음(과거 백업으로 존재). 안내 불필요.

## 7. 범위 밖 (YAGNI)

- 계정 간 할 일 공유/이동 UI(요청으로 "할 일 추가"는 이미 가능). 동기화 코드 재도입. 계정 삭제/비번 변경.

## 8. 성공 기준 (검증 가능)

1. 나두혁으로 로그인 → 기존 할 일이 보인다(이전됨). 양수민으로 전환 → 빈 목록.
2. 양수민으로 할 일 추가 후, 나두혁으로 전환 → 나두혁 목록에 그 할 일이 **없다**(분리).
3. 다른 브라우저/기기에서 같은 계정으로 로그인 → 같은 할 일이 보인다(계정 동기화).
4. 로그인 화면 배경이 불투명해 뒤 할 일이 보이지 않는다.
5. 동기화 칩이 계속 보이고, 로그인 상태에서 녹색 "동기화 켜짐"을 표시한다. 클릭 시 코드 입력 없이 계정 동기화 안내만 나온다.
6. 기존 할 일 이전은 **1회만** 일어나고, 이미 데이터가 있는 계정을 덮어쓰지 않는다.
7. 메시징(요청 보내기/받기/배지/답장)·테마·통계 등 기존 기능 정상.

## 9. 가정

- '나두혁'은 사용자 본인 계정명. 마이그레이션은 그 이름 기준.
- Firebase는 인터넷 있으면 동작(설정값 파일 내장).
