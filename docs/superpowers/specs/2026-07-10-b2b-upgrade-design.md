# OneThing B2B 고도화 설계 (2026-07-10)

사용자 승인 완료. 접근 방식: **단일 `index.html` 유지 + 3단계 순차 진행**(단계별 배포·확인).

## 목표

개인용 원씽 노트를 사내(B2B)용으로 고도화:
1. 쪽지/업무 요청 두 유형의 비실시간 메시징
2. 사용자 상태 표시(프레즌스) + 알림 정책
3. 전문적·집중 지향 디자인 리뉴얼

실시간 채팅은 의도적으로 지양. 웹 푸시 알림은 이번 범위에서 제외(앱 버전에서 검토).

## 1. 디자인 리뉴얼 (미니멀 뉴트럴) — 1단계

- 공책 메타포(양쪽 페이지, 바인딩, 줄노트 배경, 손글씨 폰트 Gaegu) 제거.
- 레이아웃: 상단 헤더 + 카드 기반 2컬럼(좌: 오늘의 원씽·집중 타이머 / 우: 할 일 목록).
- 색상:
  - 라이트: 바탕 `#f7f8fa`, 글자 `#1f2430`
  - 다크: 바탕 `#16181d`, 글자 `#e4e7ee`
  - 포인트 컬러: 차분한 남색 `#3b5bdb` 계열 단일
  - 기존 sky/forest 테마는 라이트/다크 2개로 통합
- 폰트: Pretendard 계열(시스템 고딕 폴백) 단일.
- 이모지 아이콘 → 인라인 SVG 선 아이콘 교체.
- 기존 기능 전부 유지(할 일, 우선순위 폴더, 드래그, 타이머, 검색, 언어, PIN 로그인) — 겉모습만 교체.

## 2. 상태 표시 (프레즌스) — 2단계

- 상태 3종: `focus`(원씽 중·초집중) / `work`(업무 중) / `away`(자리비움).
- 헤더에 내 상태 버튼(색 점 + 상태명), 클릭으로 변경.
- 저장: Firebase `status/{userId}` = `{ state, ts }`.
- 타이머 연동: 집중 타이머 시작 → 자동 `focus`, 종료/중단 → 이전 상태 복원.
- 다른 사용자 상태: 요청 보내기 폼의 받는 사람 옆 색 점
  (초록=업무 중, 보라=원씽 중, 회색=자리비움).
- 로그아웃/연결 끊김 시 `onDisconnect`로 자리비움 처리.

## 3. 쪽지 / 업무 요청 — 3단계

- 데이터: 기존 요청에 `kind: "memo" | "task"` 추가. kind 없는 기존 데이터는 쪽지로 간주(하위 호환).
- **쪽지(memo)**: 현재 폼 유지 — 받는 사람 + 짧은 텍스트(500자).
- **업무 요청(task)**:
  - 제목(필수) + 서식 에디터(contenteditable): 굵게, 밑줄, 글자색, 불릿, 표 삽입, 체크리스트
  - 첨부파일: 이미지/문서, 파일당 1MB, 건당 합계 2MB, Base64로 Realtime DB에 내장 저장
- 받은함/보낸함: "쪽지 | 업무 요청" 탭 분리, 안읽음 배지 각각 표시.
- 알림 정책:
  - 내 상태가 `focus`면 토스트·배지 갱신 없이 조용히 쌓임
  - 상태가 `focus`에서 벗어나는 순간 쌓인 알림을 몰아서 표시
  - 그 외 상태에서는 즉시 토스트(현행 유지)

## 4. 데이터/보안·에러 처리

- `database.rules.json`: `status` 경로, 요청 새 필드(kind/title/html/attachments) 검증 + 첨부 크기 제한.
- 첨부 제한 초과 시 전송 전 안내·차단. 오프라인 시 기존 "인터넷 필요" 안내 유지.
- 서식 본문(html)은 표시 전 허용 태그 화이트리스트로 sanitize — 타 사용자 발신 HTML이므로 필수.

## 5. 검증

- 각 단계 완료 시 commit + push(GitHub Pages 배포) 후 라이브에서 확인.
- 2개 계정으로 쪽지/업무 요청 왕복 테스트, `focus` 상태 알림 억제 확인.
- 기존 데이터(할 일, 기존 요청) 하위 호환 확인.

## 결정 이력

- 첨부파일: Firebase Storage 대신 소용량 Base64 DB 내장(무료·설정 불요).
- 용어: "쪽지 / 업무 요청".
- 에디터: 표·체크리스트까지 포함.
- 상태 전환: 수동 + 타이머 연동.
- focus 중 수신: 조용히 쌓아두기 → 상태 복귀 시 몰아서 알림.
- 디자인: 미니멀 뉴트럴(Notion·Linear 톤), 라이트/다크 유지.
