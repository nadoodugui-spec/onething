# 요청 UI 정리(인박스 모달 + 배지 + 알림) 설계

작성일: 2026-06-25
대상 파일: `C:\AI\OneThing\index.html`
선행: 요청 메신저 + 계정별 할 일 구현 이후의 UI 개선.

---

## 1. 배경 / 목적

받은/보낸 요청이 노트 아래에 **항상 카드로 펼쳐져** 화면이 지저분하다. 평소엔 깨끗하게,
요청은 **필요할 때 모달로** 보고, 새 요청은 **토스트 + 안 읽은 개수 배지**로 알리고, 열어 보면 **자동 읽음**.

## 2. 확정된 결정 (사용자 합의)

1. **컨테이너**: 노트 아래 ‘요청 띠’ 제거. 상단 도구줄에 **📥 버튼(안 읽은 개수 배지)** → 누르면 **요청 모달**.
2. **모달 구성**: 상단 탭 `[받은 N] [보낸]`. 카드 내용·버튼은 기존 그대로(자리만 모달로 이동).
3. **알림**: 앱 사용 중 새 요청 도착 시 **토스트** `🔔 {보낸사람}님이 요청을 보냈어요` + 배지 +1. (브라우저 시스템 알림은 안 씀.)
4. **자동 읽음**: ‘받은’ 탭을 열면 안 읽은 요청이 자동 읽음 → 배지 0.
5. **데이터 구조·전송·할일추가/완료/답장/삭제 기능은 변경 없음.**

## 3. UI 변경

### 3.1 제거
- `notes/...` 아래 `<section class="requests-zone">` 띠와 관련 CSS(`.requests-zone`, `.rq-col`, 2단 그리드) 제거.

### 3.2 상단 도구줄(`.tools`)
- 기존 `✉️ 요청 보내기` 버튼 유지.
- **📥 인박스 버튼**(`data-act="inbox"`) 추가 — 안 읽은 개수 배지(`unreadBadge`)를 이 버튼에 표시(현재 `userChip` 안의 배지는 이쪽으로 이동).
- 로그인 상태에서만 두 버튼 표시.

### 3.3 요청 모달 (신규 `reqModal`)
- 기존 모달 패턴(`.modal` / `.modal-box` / `.modal-head` / `.modal-close`) 재사용.
- 헤더: `✉️ 요청` + 닫기.
- 탭 버튼 2개: `받은 N` / `보낸` (`.rq-tab`, 활성 표시). 기본 활성 = 받은.
- 본문: 받은 목록(`rqInboxList`)과 보낸 목록(`rqSentList`) 중 활성 탭만 표시.
- 카드: 기존 `buildReqCard` 재사용(보낸사람/받는사람·시각·상태·내용·답장 + 버튼들). 빈 상태 문구 유지.

## 4. 동작 / 로직

### 4.1 렌더
- `renderRequests()`: 기존 로직 유지하되 대상 DOM이 모달 안의 `rqInboxList`/`rqSentList`. 탭 카운트(`받은 N`) 갱신, 활성 탭에 따라 표시/숨김.
- 모달이 닫혀 있어도 `renderRequests()`는 안전하게 동작(요소 존재).

### 4.2 새 요청 토스트
- 전역 `knownReqIds`(Set). requests 리스너가 fire될 때:
  - 첫 fire는 **초기화만**(기존 요청들을 known에 등록, 토스트 없음).
  - 이후 fire에서 `to == 나 && from != 나 && !knownReqIds.has(id) && hiddenFor에 나 없음`인 새 요청 발견 → `toast(t("rq_new", fromName))`. 모두 known에 추가.
- 배지(`renderUnreadBadge`)는 기존대로 `to==나 && status=="sent"` 개수.

### 4.3 인박스 열기 + 자동 읽음
- `openInbox()`: 모달 열고 기본 탭(받은) 활성 → `markInboxRead()` 호출 → 받은 요청 중 `status=="sent"`를 모두 `read`로(파이어베이스 업데이트) → 배지 0.
- 탭 전환 `setReqTab("inbox"|"sent")`: 활성 탭 표시 전환. 받은 탭 활성화 시 `markInboxRead()`.

### 4.4 표시/숨김
- `updateLoginGate()`: 로그인 시 `📥`·`✉️` 버튼 표시, 미로그인 시 숨김. (기존 `requestsZone` 토글 제거.)
- 미로그인/모달 닫힘 시에도 오류 없게 `if(el)` 가드.

## 5. i18n (추가)
- `rq_modal_head`: "✉️ 요청" / "✉️ Requests"
- `rq_tab_inbox`: "받은" / "Received"  (탭; 개수는 코드로 ` N` 붙임)
- `rq_tab_sent`: "보낸" / "Sent"
- `rq_new`: "🔔 {0}님이 요청을 보냈어요" / "🔔 {0} sent you a request"
- `rq_inbox_title`(인박스 버튼 title): "받은 요청" / "Inbox"
- 기존 `rq_inbox_empty`/`rq_sent_empty`/상태·버튼 문구 재사용.

## 6. 예외 / 엣지
- 새 기기 첫 로드: 기존 요청들로 토스트 폭주 방지 → 첫 리스너 fire는 초기화만.
- 모달 열어둔 채 새 요청: 토스트 + 목록 갱신. 받은 탭이 활성이면 자동 읽음 처리.
- 오프라인/DB 잠김: 읽음 처리(write) 실패해도 화면은 동작(배지는 로컬 계산이라 표시됨).

## 7. 범위 밖 (YAGNI)
- 브라우저/푸시 알림, 알림음, 요청 검색·정렬, 보관함. (추후 필요시.)

## 8. 성공 기준 (검증 가능)
1. 평소 화면(노트 아래)에 요청 카드가 안 보인다(깨끗).
2. 상단 `📥` 버튼에 안 읽은 개수가 정확히 뜬다(없으면 숨김).
3. `📥` 클릭 → 모달, `받은`/`보낸` 탭 전환 동작.
4. 새 요청 도착 시 토스트 `🔔 …님이 요청을 보냈어요` + 배지 증가.
5. ‘받은’ 탭 열면 자동 읽음 → 배지 0.
6. 카드의 할일추가/완료/답장/삭제, 보내기 모두 기존대로 동작.
7. 🌐 전환 시 새 문구 번역. 모바일에서도 모달 정상.
8. 기존 할 일·계정·동기화 기능 회귀 없음.

## 9. 가정
- 모달은 기존 모달 CSS로 모바일 대응됨.
- 배지·토스트는 앱이 열려 있을 때만(웹 특성).
