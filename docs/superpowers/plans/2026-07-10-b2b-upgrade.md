# OneThing B2B 고도화 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 개인용 원씽 노트를 사내용으로 고도화 — 미니멀 뉴트럴 디자인, 사용자 상태 표시(프레즌스), 쪽지/업무 요청 2유형 메시징.

**Architecture:** 단일 `index.html`(현재 ~2,065줄, HTML+CSS+JS 인라인) 유지. Firebase Realtime DB(`users`, `requests`에 `status` 추가) + 익명 인증. 3개 태스크 순차 진행, 태스크마다 commit + `git push origin main`(GitHub Pages 배포) 후 라이브 확인.

**Tech Stack:** Vanilla JS, Firebase compat SDK 10.12(app/auth/database), GitHub Pages. 테스트 프레임워크 없음 — 각 태스크는 브라우저 수동 검증(라이브 URL, 2개 계정)으로 확인한다.

**Spec:** `docs/superpowers/specs/2026-07-10-b2b-upgrade-design.md`

## Global Constraints

- 실시간 채팅 아님. 웹 푸시 알림은 범위 외.
- 첨부파일: 파일당 1MB, 요청당 합계 2MB, Base64로 RTDB에 내장.
- 용어(한/영): 쪽지/Memo, 업무 요청/Work Request. 상태: 원씽 중(초집중)/Focusing, 업무 중/Working, 자리비움/Away.
- 기존 데이터 하위 호환: `kind` 없는 요청은 쪽지로 취급. 기존 할 일·히스토리·사용자 데이터 마이그레이션 불필요(그대로 동작해야 함).
- 기존 기능 전부 유지: 할 일 CRUD·드래그·우선순위 폴더, 원씽 지정, 집중 타이머, 검색, 한/영 언어, PIN 로그인, 동기화, 백업, 휴지통, 통계.
- 다른 사용자가 보낸 서식 HTML은 표시 전 반드시 sanitize(화이트리스트 방식).
- 각 태스크 완료 시 commit 후 `git push origin main` (프로젝트 CLAUDE.md 규칙).

### Task 2: 상태 표시 (프레즌스)

**Files:**
- Modify: `index.html` — 헤더에 상태 버튼 UI, i18n 문자열, JS(타이머 start/stop 함수와 `initMessaging()` 부근)
- Modify: `database.rules.json` — `status` 경로 규칙

**Interfaces:**
- Consumes: Task 1의 CSS 변수·헤더 자리, 기존 `msgDb`, `currentUser`, `usersCache`, 타이머 시작/종료 지점(`timerRunning = true` 설정부·타이머 종료/리셋부).
- Produces: `myStatus()` → `"focus"|"work"|"away"` (Task 3의 알림 억제가 호출), `statusCache` = `{ userId: {state, ts} }`, `setMyStatus(state, {auto})`, `renderStatusDot(userId)` → 색 점 DOM 요소(Task 3의 받는사람 표시가 호출). DB 경로 `status/{userId} = { state: "focus"|"work"|"away", ts: number }`.

- [ ] **Step 1: 데이터 계층.** JS에 추가:

```js
let statusCache = {};                 // { userId: {state, ts} }
let prevManualStatus = "work";        // 타이머 자동 전환 전 상태(복원용)
const STATUS_KEY = "onething-status"; // 내 마지막 수동 상태 기억

function myStatus() {
  const s = currentUser && statusCache[currentUser.id];
  return (s && s.state) || "work";
}
function setMyStatus(state, opts) {
  if (!currentUser || !msgDb) return;
  if (!opts || !opts.auto) { prevManualStatus = state; localStorage.setItem(STATUS_KEY, state); }
  msgDb.ref("status/" + currentUser.id).set({ state, ts: Date.now() });
}
```

`initMessaging()`에 `msgDb.ref("status").on("value", snap => { statusCache = snap.val() || {}; renderMyStatus(); });` 추가. 로그인 성공 시(`resolveCurrentUser` 후 로그인 상태면) 저장된 `STATUS_KEY`(기본 `work`)로 `setMyStatus` 호출 + `msgDb.ref("status/"+currentUser.id).onDisconnect().set({ state:"away", ts: firebase.database.ServerValue.TIMESTAMP })` 등록. `logoutUser()`에서 away로 set 후 onDisconnect 취소.

- [ ] **Step 2: 헤더 UI.** 헤더에 상태 버튼(색 점 + 상태명) + 클릭 시 3옵션 드롭다운. 점 색: focus=`#8b5cf6`(보라), work=`var(--ok)`(초록), away=`#9aa1ad`(회색). i18n: `st_focus: {ko:"원씽 중", en:"Focusing"}, st_work: {ko:"업무 중", en:"Working"}, st_away: {ko:"자리비움", en:"Away"}`. `renderMyStatus()`가 버튼 라벨·점 색 갱신. `renderStatusDot(userId)`는 `statusCache[userId]` 기준 `<span class="st-dot st-focus|st-work|st-away">` 반환.

- [ ] **Step 3: 타이머 연동.** 타이머 시작부(`timerRunning = true; requestNotify();` 지점)에 `prevManualStatus = myStatus() === "focus" ? prevManualStatus : myStatus(); setMyStatus("focus", {auto:true});` 추가. 타이머 종료·중단·리셋 지점에 `if (myStatus() === "focus") setMyStatus(prevManualStatus || "work", {auto:true});` 추가(수동으로 focus 선택한 경우도 복원되는 것은 허용 — 단순함 우선).

- [ ] **Step 4: DB 규칙.** `database.rules.json`의 rules에 추가(기존 구조에 맞춰 병합):

```json
"status": {
  "$uid": {
    ".read": "auth != null",
    ".write": "auth != null",
    ".validate": "newData.hasChildren(['state','ts']) && newData.child('state').val().matches(/^(focus|work|away)$/)"
  }
}
```

- [ ] **Step 5: 수동 검증.** 두 브라우저(계정 A/B)로: A가 상태 변경 → B의 보내기 폼에서 A 옆 점 색 변경 확인(점 표시는 Task 3에서 폼에 붙지만, 이 시점엔 헤더 자기 상태·DB 값으로 확인). 타이머 시작→focus 자동 전환, 종료→복원. 탭 닫기→몇 초 후 away.

- [ ] **Step 6: Commit + 배포.**

```bash
git add index.html database.rules.json
git commit -m "사용자 상태 표시 추가(원씽 중/업무 중/자리비움) — 타이머 연동, onDisconnect"
git push origin main
```

- [ ] **Step 2: 첨부.** `<input type="file" multiple>` → FileReader로 dataURL 변환. 파일당 1MB(`file.size > 1048576`이면 토스트로 거부), 합계 2MB 초과 시 거부. `attachments: [{name, type, size, data}]`. 수신 카드에서 이미지는 인라인 미리보기(`<img>`), 그 외는 `<a download href=data>` 다운로드 링크.

- [ ] **Step 3: sanitize.** 표시 직전에 적용:

```js
const OK_TAGS = new Set(["B","STRONG","U","EM","I","P","BR","DIV","SPAN","UL","OL","LI","TABLE","TR","TD","TH","TBODY","FONT","INPUT"]);
function sanitizeHtml(html) {
  const tpl = document.createElement("template"); tpl.innerHTML = html || "";
  (function walk(node) {
    [...node.children].forEach((el) => {
      if (!OK_TAGS.has(el.tagName) || (el.tagName === "INPUT" && el.type !== "checkbox")) { el.replaceWith(...el.childNodes); return; }
      [...el.attributes].forEach((a) => {
        const keep = (el.tagName === "FONT" && a.name === "color") ||
                     (el.tagName === "SPAN" && a.name === "style" && /^color:\s*[#a-z0-9(),\s]+;?$/i.test(a.value)) ||
                     (el.tagName === "INPUT" && ["type","checked","disabled"].includes(a.name)) ||
                     a.name === "class";
        if (!keep) el.removeAttribute(a.name);
      });
      if (el.tagName === "INPUT") el.disabled = true;
      walk(el);
    });
  })(tpl.content);
  return tpl.innerHTML;
}
```

- [ ] **Step 4: 전송·표시.** `sendRequest()`가 kind에 따라 저장(쪽지=기존 필드, 업무 요청=+`title`,`html`(sanitizeHtml 적용 후 저장),`attachments`). `renderRequests()`의 받은함/보낸함 각각에 `쪽지 | 업무 요청` 서브탭과 개별 안읽음 배지(쪽지 배지·업무 요청 배지). `kind` 없는 기존 레코드는 `memo` 취급. `buildReqCard()`: task면 제목 굵게 + 본문 `innerHTML = sanitizeHtml(r.html)` + 첨부 목록. 보내기 폼 받는사람 select 옆에 선택된 사용자의 `renderStatusDot()` + 상태명 표시(focus면 "지금 초집중 중" 안내문).

- [ ] **Step 5: 알림 정책.** `notifyNewRequests()` 수정: `myStatus() === "focus"`면 새 요청 id를 `pendingNotifyIds` 배열에 쌓고 토스트·배지 갱신 생략(배지는 focus 해제 시 일괄 갱신 — `renderUnreadBadge()` 호출을 보류 플래그로 제어). 상태가 focus에서 다른 값으로 바뀌는 순간(`setMyStatus` 내부) `flushPendingNotifies()` 호출 — 쌓인 건수를 `"🔔 집중하는 동안 요청 N건이 도착했어요"` 형식 토스트 1개로 표시하고 배지 갱신.

- [ ] **Step 6: DB 규칙.** `database.rules.json`의 requests 항목에 검증 추가: `kind`는 `memo|task`(없어도 허용), `title` 200자 이하, `html` 200KB 이하, attachments 각 항목 `data` 길이 1.5M 문자 이하(Base64 팽창 감안).

- [ ] **Step 7: 수동 검증.** 두 계정으로: 쪽지 왕복, 업무 요청(서식+표+체크리스트+이미지 첨부) 왕복, 탭·배지 분리 확인, 1MB 초과 파일 거부, 기존(구버전) 요청이 쪽지 탭에 그대로 보이는지, B가 focus일 때 A가 보내면 B에 토스트 없음 → B가 work로 바꾸면 몰아서 토스트. `<img src=x onerror=alert(1)>`를 본문에 붙여넣어 전송 → 수신측에서 스크립트 실행 안 됨 확인.

- [ ] **Step 8: Commit + 배포.**

