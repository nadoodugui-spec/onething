관리자 권한 흐름도

최고관리자 전용 문서 · 위에서 아래로 권한이 좁아집니다 · 모든 권한은 서버 보안 규칙으로 강제됨

👑 1. 최고관리자
플랫폼 운영자 — 나두혁 (nadoodugui@gmail.com) · 계정 1개, 코드에 고정
  • 플랫폼 관리 화면: 전체 워크스페이스(회사)·회원 수 요약, 회사별 구성원 목록, 이름/이메일 실시간 검색
  • 계정(프로필) 삭제 — 시스템 전체에서 유일한 삭제 권한 (본인 계정은 삭제 불가 처리)
  • 비밀번호 재설정 메일 발송 — 본인 셀프 재설정이 안 될 때의 최종 지원 창구
  • 모든 회원의 프로필(이름·이메일) 수정 가능
  • 보안 규칙 배포, 시스템·서버 설정, 데이터 정리 등 운영 전권
  • + 아래 모든 등급의 기능 사용 가능(본인도 한 회사의 오너)
▼ 임명 (아직 도입 전)
🛡 2. 웹사이트 관리매니저
최고관리자가 지정하는 플랫폼 운영 보조 등급 · 여러 명 가능 · 소속 회사와 무관
  • 임명/해제 방법: 최고관리자가 관제 센터(admin.html)의 회원 행에서 [매니저 지정]/[매니저 해제] 버튼으로. 별도 가입 절차 없이 기존 회원 누구나 지정 가능하며, 즉시 적용됩니다.
  • 접속 방법: 매니저 본인 계정으로 앱에 로그인한 상태에서 admin.html 주소를 열면 "관리매니저 모드"로 열립니다(위험 버튼은 화면에서 아예 사라짐).
  • 조회 권한: 전체 현황 숫자(회사·회원·신규 가입), 주간 가입 추이, 회원 구성, 회사별 카드·구성원 목록·상세 통계(요청 완료율·프로젝트·상태 분포), 검색.
  • 회원 지원 권한: 비밀번호 재설정 메일 발송, 문의(Q&A) 답변 작성·수정 — 회원이 앱의 "문의하기"로 남긴 질문에 답하면 그 회원의 문의 내역에 매니저 이름으로 표시됩니다.
  • 일반 회원으로서의 활동: 매니저 지정과 무관하게 자기 회사의 원씽·요청 등 일반 기능은 평소대로 사용합니다.
✗ 불가: 계정 삭제, 회사 삭제, 회원 이름 변경, 다른 매니저 지정/해제, 문의 삭제, 보안 규칙·시스템 설정 — 전부 최고관리자 전용이며 화면이 아니라 서버 규칙이 거부합니다. 매니저는 각 회사의 팀 데이터를 조회(통계)할 수만 있고 수정할 수 없습니다.
▼ 회사(워크스페이스)마다 아래 구조가 반복
🏢 3. 오너 (오너관리자)
회사(워크스페이스)를 만든 사람 · 회사당 1명
  • 관리자 승격/해제 — 구성원을 사내관리자로 올리고 내림 (화면·서버 규칙 모두 오너 전용)
  • 워크스페이스 정보(이름 등) 관리
  • + 사내관리자의 모든 권한 포함
✗ 불가: 구성원의 프로필(이름·이메일) 수정, 비밀번호 관련, 계정 삭제, 다른 회사 데이터 접근
▼ 승격
💼 4. 사내관리자
오너가 승격한 구성원 · 여러 명 가능
  • 초대: 초대 코드(6자리) 확인·재발급 — 유출 시 재발급으로 무효화
  • 내보내기: 구성원의 소속 해제(계정·개인 데이터는 그대로) — 미완료 요청은 나에게 자동 전달, 6초 실행 취소
  • 팀 통계(통계의 '팀' 탭) 열람
  • 관리 기록(감사 로그) 열람 — 모든 관리 행위가 기록됨
✗ 불가: 승격/해제(오너 전용), 구성원 프로필 수정, 비밀번호 관련, 계정 삭제
▼ 초대 코드로 참여
👤 5-A. 일반 구성원
회사에 소속된 가입자
  • 개인 기능 전부: 원씽·할 일(Now/Next/Routine)·List·타이머·통계·백업
  • 팀 기능 전부: 상태 표시, 쪽지/업무 요청(보내기·수정·회수·전달·진행), 프로젝트 칸반, 팀 현황
  • 내 프로필(이름)·내 비밀번호·내 공개 설정은 본인만 관리
🙋 5-B. 무소속 가입자
회사에 참여하지 않은 가입자
  • 개인 기능은 전부 사용 가능
  • 팀 기능은 잠김 — 초대 코드 참여 또는 회사 만들기(→오너)로 열림

설계 원칙 3가지
① 계정은 개인의 것, 소속은 회사의 것 — 회사는 초대·내보내기만, 계정·비밀번호·프로필은 본인(+최고관리자)만.
② 직책은 오너만 — 사내관리자가 세력을 불릴 수 없음.
③ 회사 간 완전 격리 — 각 회사의 요청·상태·프로젝트는 그 회사 구성원 외에는 서버가 읽기조차 거부.

운영 시나리오 빠른 참조

  • 구성원이 비밀번호를 잊음 → 본인이 로그인 화면 "비밀번호 찾기" → 안 되면 최고관리자가 플랫폼 관리에서 재설정 메일 발송
  • 퇴사 처리 → 사내관리자가 "내보내기"(소속만 해제, 미완료 업무는 관리자에게 인계)
  • 계정 완전 삭제 요청 → 최고관리자가 플랫폼 관리에서 삭제 (소속 여부와 무관하게 가능)
  • 초대 코드 유출 → 사내관리자/오너가 코드 재발급 (기존 코드 즉시 무효)
  • 회원 문의 접수 → 회원이 앱 "문의하기(Q&A)"에 작성 → 관제 센터 문의 패널에서 최고관리자/관리매니저가 답변 → 회원의 문의 내역에 표시
  • 사내관리자가 말썽 → 오너가 해제 → 필요 시 내보내기 → 심각하면 최고관리자가 계정 삭제
